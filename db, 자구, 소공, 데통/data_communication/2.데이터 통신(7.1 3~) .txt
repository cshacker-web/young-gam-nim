동기식 전송 VS 비동기식 전송

데이터 통신에서는 동기식 전송을 사용한다고 보면된다.
이유는 비동기식 전송은 시작비트 문자비트(7~8bit) 정지비트 와 같은 형태로
오버헤드가 크고 효율성이 낮으며 데이터 전송속도가 느린 단점을 가지고 있기
때문이다. 비동기식은 한번에 한 문자씩 전송한다.
동기식 전송은 동기화를 클럭 신호로 송, 수신측이 클럭을 맞춰 통신하며
프레임(1바이트) 형태로 데이터를 전송한다.

동기식 전송 방식
바이트기반 전송 VS 비트 기반 전송
바이트 기반 : BISYNC, PPP, DDCMP
각 프레임을 바이트의 연속으로 간주

비트 기반 : Ethernet, HDLC(High-level Data Link Contorl)
각 프레임을 비트의 연속으로 간주
프레임의 시작과 끝을 알리기 위해 플래그라고 하는 특수한 비트 패턴
("01111110") 사용

비트 기반 문제점
데이터 필드에 플래그 패턴(01111110)이 존재하는 경우
비트 스터핑 : 송신시 5개의 1을 발견하면 0을 추가로 삽입, 수신시 스터핑 제거

바이트 기반에서도 바이트 스터핑이 필요하다.
8bit STX(Start of Text) + 데이터 + 8bit ETX(End of Text) 구조에서
데이터가 ETX와 같은 경우 앞에 DLE(Data-Link Escape)이라는 문자를 추가

교환 기술
1. 회선 교환 (Circuit Switching)
(공중전화) 회선 설정->데이터 전송->회선 해제
채널의 대역폭을 독점한다는 단점이 있지만 신뢰성 있는 데이터를 전송하고
일정한 데이터 전송률로 전송된다는 장점이 있음
회선 설정 시간이 필요하고 링크 및 스위칭 장비가 성능에 영향을 준다.

다중화(Multiplexing) 기술 (대역폭을 채널로 분할)
다수의 송신 호스트가 보내는 데이터를 하나의 전송매체를 통해 전송
전송매체가 갖고 있는 대역폭을 다수의 채널로 분할,
각각의 호스트가 하나의 채널을 사용하도록 하는 기술

TDM (디지털 신호는 보통 이 방식)
전송매체의 대역폭을 타임 슬롯으로 분할
각 호스트는 할당된 타임 슬롯을 사용하는 방식

FDM
전송매체의 대역폭을 주파수로 분할
각 호스트는 할당된 주파수를 사용하는 방식, 아날로그 신호에 사용

2. 메시지 교환 (Message Switching)
편지, 메시지 논리적인 단위로 전송, 메시지 헤더에 목적지 주소 표시
송신, 수신간 미리 경로 설정 필요 X
각 노드는 메시지를 전부 받을때까지 '저장'한 후 경로상의 또 다른 노드로 전송
(Store - And - Forward)
한 개의 메시지 복사, 여러 목적지로 보낼 수 있다는 장점
메시지 우선 순위 가능, 메시지 단위 오류 제어 가능
메시지가 노드에 저장되기 때문에 지연 발생, 실시간 통신에서 어려움

3. 패킷 교환 (Packet Switching)
회선 교환 + 메시지 교환의 장점만 모음
회선 교환 VS 가상회선 패킷교환 VS 데이터그램 패킷교환
(최초 물리적 회선 시간) VS (최초 논리적 회선 시간) VS (없음)

가상회선(Virtual Circuit) 패킷교환 - ATM
하나의 물리적인 회선을 가지고 있더라도 여러개의 논리적인 회선으로
가상으로 회선을 설정하여 패킷을 교환
링크 = 라우터 4분면(ex)의 번호
VCI 번호 = 라우터 테이블에서 기록된 패킷을 구분하기 위한 패킷내의 번호

데이터그램 패킷 교환 - 인터넷
연결 설정 및 해제 과정이 없음
보낸 순서와 다르게 수신측에 도착할 수 있어 수신측에서 재정렬해야 함
적은양의 데이터를 신속하게 보낼때 유리하다.
목적지에 따라서 링크의 비율이 라우팅 테이블에 적혀있다.
A에서 C로 바로가는 패킷과 B를 거쳐서 가는 패킷이 있기때문에 C에서는
순서를 보장할 수 없다.
